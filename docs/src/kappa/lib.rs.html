1//! This tool produces complex dust particle opacities right from the command line. It is
2//! derived from Michiel Min’s DHS OpacityTool and also implements Ryo Tazaki’s MMF
3//! theory for highly porous aggregates.
4//!
5//! # Capabilities
6//! - stand-alone tool, fully command line driven, no input files need to be edited
7//! - full scattering matrix output in several formats, including for RADMC-3D
8//! - combining materials through mixing into a complex grain with porosity
9//! - built-in: a curated collection of materials for applications in astronomy
10//! - external refractive index data can be used just as easily
11//! - computational methods: (i) DHS (Distribution of Hollow Spheres) for irregular
12//!   grains and low-porosity aggregates. Standard Mie theory for perfect spheres
13//!   is available as a limiting case. (ii) MMF (Modified Mean Field) theory for
14//!   high-porosity/fractal aggregates. (iii) CDE approximation in the Rayleigh limit.
15//! - Python interface module for plotting and post-processing
16//!
17//! # Physical units used by `kappa`
18//! Due to conventions in our field, the input and output of `kappa` uses the following units
19//!
20//! - grain sizes and wavelengths are in microns
21//! - mass densities of materials are in g/cm^3
22//! - opacities are in cm^2/g
23//! - scattering matrices are in sr^-1 or cm^-1 g^-1 sr^-1
24//!
25//! # Examples
26//! A simple grain made only of the default pyroxene, for the default grain size distribution
27//! ($a^{-3.5}$ powerlaw from 0.05 to 3000μm), on the default wavelength grid (0.05μm to 1cm).
28//!
29//! # Installation
30//! The easiest way to install `kappa` is to first get the source code from GitHub:
31//! ```bash
32//! git clone https://github.com/ProsiaLAB/kappa.git
33//! ```
34//! Then, you can build the executable with `cargo`:
35//! ```bash
36//! cd kappa
37//! cargo build --release
38//! ```
39//! This will create a binary called `kappa` in the `target/release` directory.
40//!
41//! # Usage
42//! The command line interface is documented in the help message:
43//! ```bash
44//! kappa --help
45//! ```
46//!
47//! # Material properties
48//! `kappa` needs refractive index data to work.  For your convenience, a
49//! useful list of materials is compiled into `kappa`. You can also find
50//! and use other data.
51//! To access one of the built-in materials, specify the corresponding key
52//! string like `pyr-mg70`. In each material class we have selected a
53//! useful default, accessible with an even simpler generic key (for
54//! example, `pyr` is an alias for `pyr-mg70`). Most of the built-in
55//! refractive index datasets have a reasonably wide wavelength coverage -
56//! the few exceptions are highlighted by bold-face numbers.  If a
57//! material is being used outside of the measured region, `kappa` will
58//! still function, using extrapolated optical properties.
59//!
60//! Even the limited number of materials we have selected to include with
61//! `kappa` can be daunting. To get started with some kind of standard
62//! opacity, we recommend to work with pyroxene `pyr`, carbon
63//! \fbox{c}, and, at low temperatures, water ice `h2o` (Woitke+
64//! 2016). If you need to account for sulfur, you may want to include
65//! troilite `tro` (Birnstiel+ 2016).
